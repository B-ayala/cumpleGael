# 05 — UX / PRODUCTO

## Principios
- **Simplicidad sobre todo.** Menos clicks, menos campos, menos pantallas.
- **Feedback visual inmediato** en cada acción: loading spinner, toast de éxito, mensaje de error claro.
- **Confirmación** en acciones destructivas (eliminar, cancelar, cerrar sesión, etc.).
- **Errores en lenguaje humano**, no códigos crudos. "No pudimos guardar tus cambios, reintentá" > "Error 500".
- **Validación inline** en formularios, no recién al submit.
- **Empty states con CTA** (qué puede hacer el usuario ahora).
- **Jerarquía visual** clara: una acción primaria por pantalla.
- **Pensar en el usuario real**, no en el desarrollador. Si tu abuela no puede usarlo, está mal diseñado.

## Microinteracciones obligatorias
- Botones con estado `hover`, `active`, `disabled`, `loading`.
- Inputs con foco visible, error visible, hint cuando ayude.
- Skeleton loaders > spinners cuando carga contenido (menos percepción de espera).
- Animaciones cortas (150–250ms), nunca bloqueantes.

## Formularios
- Labels siempre presentes (no solo placeholders).
- Mensajes de error específicos por campo, no genéricos.
- Autofocus en el primer campo.
- Submit con Enter funciona.
- Estado de submit deshabilita el botón para evitar dobles envíos.

## Navegación
- El usuario siempre sabe **dónde está** (breadcrumbs / título claro).
- Siempre puede **volver** sin perder datos.
- Botón primario fácilmente identificable (color, posición, tamaño).

## Contenido
- Tono consistente (vos / tú / usted, elegir uno).
- Evitar jerga técnica de cara al usuario final.
- Textos cortos. Si necesita explicación larga, el flujo está mal diseñado.

## Test de los 5 segundos
Si en 5 segundos un usuario nuevo no entiende qué hace la pantalla y qué se espera de él → rediseñar.
